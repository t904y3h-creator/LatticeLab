# CLI

`CLI` — это консольный вход в движок `LATTICE`.

Сейчас это минимальный интерфейс без GUI поверх движка. Он уже полезен для технических сценариев, но как полноценная рабочая оболочка он еще требует развития.

## Роль

`CLI` нужен для случаев, когда не хочется поднимать UI и нужен быстрый доступ к движку из терминала.

Типичные сценарии:

- вручную прогнать несколько шагов симуляции
- посмотреть позиции атомов
- вывести базовые метрики
- быстро проверить поведение ядра вне desktop-приложения

## Перспектива

Наиболее интересное направление для CLI — использование движка без GUI вне desktop-приложения.

В перспективе он хорошо подходит для таких сценариев:

- запуск симуляций на серверах без GUI
- batch-прогоны наборов сцен
- автоматизированные тестовые и исследовательские сценарии
- интеграция с внешними пайплайнами
- удаленный запуск и сбор результатов

То есть долгосрочно CLI может быть не просто "консольной демкой", а рабочей точкой входа для автоматизации и серверного запуска симуляции.

Сейчас CLI:

- парсит аргументы командной строки через `CLI11`
- создает `Simulation`
- поднимает простую стартовую сцену
- заходит в цикл чтения команд из stdin

## Что Поддерживается Сейчас

Аргументы запуска:

- `--steps`
- `--dt`
- `--gravity`
- `--world-size`
- `--max-speed`
- `--damping`
- `-v`, `--verbose`

Команды в интерактивном режиме:

- `step`
- `run [count]`
- `positions`
- `metrics`
- `reset`
- `help`
- `quit`

Это уже позволяет использовать CLI как отладочный интерфейс поверх `Simulation`.

## Что Можно Улучшить

Если развивать CLI дальше, логичными следующими шагами были бы:

- расширить набор команд
- добавить работу со сценами и состояниями
- добавить команды для конфигурации мира и физических параметров
- добавить batch-режим или сценарный режим
- сделать вывод более структурированным для автоматизации
